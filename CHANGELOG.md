# Changelog

All notable changes to Open Satchel. Follows [Keep a Changelog](https://keepachangelog.com/).

## [0.1.0] - 2026-04-17 — M1 scaffold

### Added
- Initial Tauri 2.x + Rust + React scaffold.
- Rust IPC layer: file open/save, folder walk, hash, recent files, app version.
- PDF viewer via pdfjs-dist with thumbnail sidebar.
- Zustand state layer (tab, format, UI, history).
- Format registry pattern with `DocumentFormat` taxonomy.
- Global keyboard shortcuts.
- AGPL-3.0 license intent (install the full text via `gh api /licenses/agpl-3.0 --jq .body > LICENSE`).

### Changed
- Complete rewrite from the previous Electron implementation. The archived
  Electron codebase (v1.0, ~93% Acrobat parity) lives at
  [JayQuan-McCleary/open-satchel-electron-archive](https://github.com/JayQuan-McCleary/open-satchel-electron-archive)
  as reference material for the port.

### Deferred from v1 (see DEFERRED_FORMATS.md)
- 25 formats dropped from v1 scope to focus on PDF + Office parity.
- ~20 PDF features deferred to M2–M4 (text editing, annotations, forms,
  redaction, signatures, OCR, compare, bookmarks, page manager, conversion).

### Known limitations
- PDF rendering uses pdfjs in-WebView (M1 only). Native `pdfium-render` in M2.
- No MSO fidelity — LibreOffice sidecar arrives in M6.
- Only PDF format registered; 9 more formats scheduled for M5–M6.
- Placeholder Tauri icons generated by `scripts/gen-icons.ps1`; real logo pending.
